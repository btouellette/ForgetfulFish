# Web Prototype Plan (Current Card Set)

Status: active implementation plan for making the existing ruleset playable from the web UI.

## Goal

Ship a usable web prototype where two authenticated players can create/join/start a room and complete real gameplay actions for the currently implemented cards without manual DB/script intervention.

Current implemented card registry baseline (source of truth):

- `island`
- `brainstorm`
- `predict`
- `memory-lapse`
- `mystical-tutor`
- `accumulated-knowledge`

See `packages/game-engine/src/cards/index.ts`.

## Scope

In scope:

- Choice UX for pending choices generated by current rules (`CHOOSE_CARDS`, `ORDER_CARDS`, `NAME_CARD`, existing `CHOOSE_YES_NO`).
- Web commands needed to play currently implemented cards (`PLAY_LAND`, `CAST_SPELL`, `PASS_PRIORITY`, `MAKE_CHOICE`, `CONCEDE`).
- Minimal target picking flow required by current cards (notably Memory Lapse).
- End-to-end automated proof that the browser path can execute these flows.

Out of scope:

- New card/rules-engine features from Phase 3+.
- Full legal-actions manifest/protocol redesign.
- Canvas interaction rewrite or art-heavy polish.

## Prototype Exit Criteria

- Two users can complete the room join/start flow and continue into active gameplay from `/play/[roomId]`.
- Implemented card set is actually playable from browser inputs (not only via direct API/script).
- Pending choices required by implemented cards are completable in UI.
- Reconnect/resync preserves actionable gameplay state.
- Automated tests cover the playable loop and pass in CI.

## Risks and Mitigations

- Missing legal-action manifest may allow invalid client attempts.
  - Mitigation: server-authoritative validation remains source of truth; surface clear command errors in UI.
- Choice constraint drift between engine and web UI.
  - Mitigation: keep one parser module (`apps/web/lib/pending-choice.ts`) plus focused tests.
- E2E flake in realtime scenarios.
  - Mitigation: reuse deterministic fixture tokens/flows from existing `e2e/room-realtime.spec.ts`.
